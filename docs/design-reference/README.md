# Design Reference

Static HTML mockups used to validate the visual language and screen layouts before Flutter implementation. These are the source of truth for design tokens (colors, typography, spacing) — see CONTRIBUTING.md for the rule on keeping these and `lib/core/theme/` in sync.

- `app-mockups-v2.html` — Library, Player, Voices, Import sheet (v2: light palette revision)
- `app-mockups-core-batch1.html` — Settings, Book detail/Chapter list, Empty library state
- `app-mockups-core-batch2.html` — Search results, Onboarding, Sign-in
- `app-mockups-secondary-batch.html` — Voice detail/preview, Pronunciation dictionary, Download manager
- `reader-player-mockup.html` — early exploration (dark full-bleed theme) — superseded by v2, kept for historical reference

Open any file directly in a browser to view.
